# Contributing to Web3 Security Scout

Thank you for considering a contribution! This guide helps you get started.

## Code of Conduct
By participating, you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute
- Report bugs via Issues (use the Bug Report template)
- Suggest enhancements or new features
- Submit Pull Requests (follow the PR template)
- Improve documentation

## Development Setup
1. Fork the repository
2. Clone your fork
3. Install dependencies: `pip install -r requirements.txt`
4. Run tests: `pytest`
5. Create a branch: `git checkout -b feature/your-feature`
6. Commit and push
7. Open a PR

## Style Guide
- Follow PEP8 for Python
- Write docstrings for functions and classes
- Keep changes focused; avoid unrelated refactors

## Commit Messages
Use conventional commits: `feat:`, `fix:`, `docs:`, `chore:`, etc.

## Review Process
- CI must pass
- At least one maintainer approval required
- address feedback promptly
